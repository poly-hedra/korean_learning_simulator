### TODO 04 — 전체 레이아웃 컴포넌트
`app/layout.tsx`를 수정하여 모든 페이지에 공통 적용되는 모바일 고정 레이아웃을 만들었습니다.

- `max-w-[480px]` — 최대 너비 480px 제한
- `mx-auto` — 화면 중앙 정렬
- `min-h-screen` — 최소 높이 = 화면 전체
- `lang="ko"` — HTML 언어 속성 한국어로 변경

홈 페이지(`page.tsx`)도 기본 보일러플레이트를 정리하고, 디자인 토큰이 적용된 임시 화면으로 교체했습니다.

## 왜 이렇게 했나요?

- **폴더 구조**: 컴포넌트, 훅, 타입을 분리하면 코드를 찾기 쉽고, 여러 페이지에서 재사용 가능
- **디자인 토큰**: CLAUDE.md에 명시된 색상을 Tailwind 시스템에 등록해두면, 나중에 색상을 바꿔야 할 때 한 곳만 수정하면 됨
- **모바일 고정 레이아웃**: 기획서 요구사항 (480px 고정). 모든 페이지에 자동 적용되어 각 페이지에서 반복 코드 불필요
- **Tailwind v4 방식**: v4부터는 `tailwind.config.ts` 대신 CSS의 `@theme` 블록에서 토큰을 정의하는 것이 표준

## 빌드 확인

`npm run build` 성공 확인 완료.

## 기술 용어 주석

- **디자인 토큰**: 색상, 폰트, 간격 등 디자인에서 반복 사용되는 값을 변수로 정의한 것. 한 곳에서 수정하면 전체에 반영됨.
- **`@theme` 블록**: Tailwind CSS v4에서 커스텀 색상/폰트를 등록하는 방법. 여기에 등록하면 `bg-gold`, `text-orange` 같은 클래스가 자동 생성됨.
- **`.gitkeep`**: Git은 빈 폴더를 추적하지 않으므로, 빈 파일을 넣어 폴더가 커밋에 포함되도록 하는 관행.
- **`mx-auto`**: Tailwind 클래스. margin-left와 margin-right를 auto로 설정하여 요소를 수평 중앙에 배치.
- **`min-h-screen`**: 최소 높이를 화면(viewport) 전체 높이로 설정하는 Tailwind 클래스.
