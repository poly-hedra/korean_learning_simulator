Kiwi 품사 태그 안내서 (프로젝트 전용)

목적
- 이 문서는 본 프로젝트에서 실제로 사용하거나 로그에서 확인한 Kiwi 품사 축약어를 한국어로 정리한 문서입니다.
- 특히 어휘 평가 단계에서 태그가 어떻게 정규화되고, `database/vocabulary.json`의 `kind`와 어떻게 연결되는지 설명합니다.

----------------------------------------
1. 가장 중요한 규칙: 활용형 태그는 기본 품사로 본다
----------------------------------------
- Kiwi는 활용형에 접미를 붙여 태그를 내놓을 수 있습니다.
  - 예: `VV-R`(규칙 활용 동사), `VV-I`(불규칙 활용 동사)
- 이 프로젝트는 `-` 앞부분만 기본 품사로 사용합니다.
  - 예: `VV-R` -> `VV`, `VA-I` -> `VA`
- 따라서 `받/VV-R`도 동사(`VV`)로 처리되어 기본형 `받다` 매칭 대상에 포함됩니다.

관련 구현 위치
- `02_evaluation/dialogue_cleansing.py`
  - `base_pos_tag()`
  - `is_main_pos_tag()`
  - `is_display_pos_tag()`

----------------------------------------
2. 어휘 매칭에 직접 쓰는 주요 품사
----------------------------------------
아래 품사는 평가 노드에서 매칭 대상(주요 품사)으로 처리됩니다.

- `NNG` (일반 명사)
  - 뜻: 보통명사
  - 예문: "호박을 샀어요." -> `호박/NNG`
  - `vocabulary.json kind` 연결: `명사`, `의존명사`

- `NNP` (고유 명사)
  - 뜻: 사람/지명/기관명 같은 고유명사
  - 예문: "서울에 가요." -> `서울/NNP`
  - `vocabulary.json kind` 연결: `명사`

- `NNB` (의존 명사)
  - 뜻: 홀로 쓰이기보다 수식이 필요한 명사
  - 예문: "먹을 거가 없어요." -> `거/NNB`
  - `vocabulary.json kind` 연결: `의존명사`, `명사`

- `NP` (대명사)
  - 뜻: 사람/사물을 대신 가리키는 말
  - 예문: "그것이 뭐예요?" -> `그것/NP`
  - `vocabulary.json kind` 연결: `대명사`

- `NR` (수사)
  - 뜻: 수량/순서를 나타내는 말
  - 예문: "두 개 주세요." -> `두/NR`
  - `vocabulary.json kind` 연결: `수사`

- `VV` (동사) + 활용형(`VV-R`, `VV-I` 등)
  - 뜻: 동작/행위를 나타냄
  - 예문: "선물을 받고 있어요." -> `받/VV-R`, lemma=`받다`
  - 정규화: lemma 우선 사용, 결과 `받다`
  - `vocabulary.json kind` 연결: `동사`

- `VA` (형용사) + 활용형(`VA-R`, `VA-I` 등)
  - 뜻: 상태/성질을 나타냄
  - 예문: "날씨가 좋아요." -> `좋/VA`, lemma=`좋다`
  - 정규화: lemma 우선 사용, 결과 `좋다`
  - `vocabulary.json kind` 연결: `형용사`

- `MAG` (일반 부사)
  - 뜻: 용언/문장 성분을 꾸미는 부사
  - 예문: "정말 맛있어요." -> `정말/MAG`
  - `vocabulary.json kind` 연결: `부사`

- `MAJ` (접속 부사)
  - 뜻: 앞뒤 문장을 이어 주는 부사
  - 예문: "그리고 집에 갔어요." -> `그리고/MAJ`
  - `vocabulary.json kind` 연결: `부사`

----------------------------------------
3. 용언 정규화에서 함께 보는 품사
----------------------------------------
아래 품사는 "기본형으로 정규화"할 때 용언 계열로 취급합니다.

- `VV` 동사
- `VA` 형용사
- `VX` 보조 용언
- `XSV` 동사 파생 접미사
- `XSA` 형용사 파생 접미사

정규화 규칙
- 해당 품사(활용형 포함)면 lemma를 우선 사용합니다.
- lemma가 있으면 보통 `...다` 형태(사전형)로 맞춰서 매칭합니다.

예시
- `받/VV-R`, lemma=`받다` -> 정규화 결과 `받다`
- `듣/VV-I`, lemma=`듣다` -> 정규화 결과 `듣다`
- `먹/VV`, lemma=`먹다` -> 정규화 결과 `먹다`

----------------------------------------
4. 로그에서 자주 보이지만 직접 매칭 대상은 아닌 품사
----------------------------------------
- `EC` 연결 어미
  - 예: "먹고"의 `고/EC`
- `EF` 종결 어미
  - 예: "먹어요"의 `어요/EF`
- `EP` 선어말 어미
  - 예: "먹었어요"의 `었/EP`
- `ETM` 관형형 전성 어미
  - 예: "먹을 것"의 `을/ETM`
- `IC` 감탄사
  - 예: "아", "음"
- `JKS` 주격 조사
  - 예: "이/가"
- `JKO` 목적격 조사
  - 예: "을/를"
- `JKB` 부사격 조사
  - 예: "에/에서/한테"
- `JX` 보조사
  - 예: "은/는/도"
- `VCP` 서술격 조사(지정사)
  - 예: "이다"
- `XSN` 명사 파생 접미사
  - 예: "사장님"의 `님/XSN`

----------------------------------------
5. vocabulary.json 품사(kind)와의 연동 방식
----------------------------------------
입력 데이터 구조
- 파일: `database/vocabulary.json`
- 주요 필드: `index`, `word`, `kind`, `example`

매칭 흐름
1) 사용자 발화를 Kiwi로 토큰화한다.
2) 토큰마다 `normalize_token_for_vocab()`을 적용한다.
   - 용언 계열은 기본형(lemma) 중심으로 정규화
3) `canonicalize_word()`로 표제어를 비교 가능한 형태로 맞춘다.
   - 예: `받다01` -> `받다`
4) 동음이의어가 있으면 `resolve_entry_by_pos()`에서 Kiwi 품사 힌트로 1차 필터링한다.
5) 최종적으로 `kind`와 `index`를 포함한 엔트리를 확정한다.

품사 힌트 테이블 (현재 기준)
- `MAG`, `MAJ` -> `부사`
- `NNG`, `NNP`, `NNB` -> `명사`, `의존명사`
- `NP` -> `대명사`
- `NR` -> `수사`
- `VV` -> `동사`
- `VA` -> `형용사`

----------------------------------------
6. 실제 케이스 예시
----------------------------------------
예문: "선물을 받고 음악을 듣고 밥을 먹고 길을 걸어가고 친구를 도와주고 있어요"

- 받
  - form: `받`
  - tag: `VV-R` (기본 품사 `VV`)
  - lemma: `받다`
  - normalized: `받다`
  - 기대 매칭: `받다01` (동사)

- 듣
  - form: `듣`
  - tag: `VV-I` (기본 품사 `VV`)
  - lemma: `듣다`
  - normalized: `듣다`

- 먹
  - form: `먹`
  - tag: `VV`
  - lemma: `먹다`
  - normalized: `먹다`

- 걸어가
  - form: `걸어가`
  - tag: `VV`
  - lemma: `걸어가다`
  - normalized: `걸어가다`

- 도와주 (분석 결과에 따라 `돕`으로 분해될 수 있음)
  - form: `돕`
  - tag: `VV-I` (기본 품사 `VV`)
  - lemma: `돕다`
  - normalized: `돕다`

7. 유지보수 메모
----------------------------------------
- 운영 중 새 태그가 로그에 나타나면 이 문서와 `TAG_KIND_HINTS`를 함께 갱신합니다.
- 품사-매칭 문제를 점검할 때는 아래를 우선 확인합니다.
  - `is_main_pos_tag()` 필터
  - `normalize_token_for_vocab()` 결과
  - `resolve_entry_by_pos()`의 kind 힌트
